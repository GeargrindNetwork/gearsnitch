# GearSnitch Product Roadmap

## Current Phase

Phase 3 execution queue after the foundation tracks closed.

## Sequencing

### Phase 1

- backend core services
- web browser sign-in and account bootstrap

### Phase 2

- iOS completion against live services
- session logging and calendar reliability

### Phase 3

- GPS route capture and replay
- realtime, worker, and notification-contract hardening

### Phase 4

- device priority alarm and favorite-device polish
- richer web dashboard analytics

### Phase 5

- production QA sweep across all three user surfaces

### Phase 6

- AirPods Pro 3 heart rate integration: real-time BPM in Dynamic Island, lock screen Live Activity, and in-app dashboard with backend persistence

### Phase 7

- Apple Watch companion app: native watchOS app with heart rate, session control, device alerts, and quick actions
- Cross-platform continuity: WatchConnectivity for iPhone-Watch sync, unified health dashboard on web, backend health dashboard endpoint
- Full device fleet visibility across iOS, Watch, and Web with health source attribution

## Follow-Up Tracks

- external deployment and store/account configuration once Apple, Google, APNs, and payment credentials are ready
- social sharing, achievements, and Apple Watch companion work after the launch-critical loop closes
- peptide/steroid cycle tracking as a shared health-adjacent logging and reporting domain after the backend contract is defined
- heart rate alerts, zone coaching, and historical HR charting after AirPods Pro 3 integration stabilizes
